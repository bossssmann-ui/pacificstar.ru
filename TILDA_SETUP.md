# Инструкция: как дать доступ к Tilda и загрузить код

## Часть 1 — Как предоставить доступ разработчику к Tilda

### Шаг 1. Войдите в аккаунт Tilda
Перейдите на [tilda.cc](https://tilda.cc) и войдите в свой аккаунт.

### Шаг 2. Откройте настройки сайта
На панели сайтов кликните на **«...» (три точки)** рядом с вашим сайтом →  
выберите **«Настройки сайта»** (Site Settings).

### Шаг 3. Добавьте соавтора
1. В меню слева выберите раздел **«Соавторы»** (Collaborators).
2. Нажмите **«Добавить соавтора»**.
3. Введите **e-mail** разработчика и выберите уровень доступа:
   - **Редактор** — может редактировать страницы, но не настройки оплаты.
   - **Владелец** — полный доступ (не рекомендуется для внешних разработчиков).
4. Нажмите **«Отправить приглашение»**.

> ✅ Разработчик получит письмо с приглашением. После принятия он сможет  
> редактировать сайт в вашем аккаунте без доступа к вашему паролю.

## Часть 2 — Как загрузить HTML/CSS/JS блоки кода вручную

### Метод A — Глобальный код (все страницы)
Подходит для подключения `style.css`, `main.js`, `i18n.js`, `map.js`, `currency.js` и т.д.

1. **Настройки сайта** → **«Ещё»** (More) → **«HTML-код»** (HTML code).
2. В поле **«Перед закрывающим тегом `</head>`»** вставьте:
   ```html
   <link rel="stylesheet" href="https://ВАШ_ДОМЕН/css/style.css">
   ```
3. В поле **«Перед закрывающим тегом `</body>`»** вставьте:
   ```html
   <script src="https://ВАШ_ДОМЕН/js/main.js"></script>
   <script src="https://ВАШ_ДОМЕН/js/i18n.js"></script>
   <script src="https://ВАШ_ДОМЕН/js/map.js"></script>
   <script src="https://ВАШ_ДОМЕН/js/currency.js"></script>
   <script src="https://ВАШ_ДОМЕН/js/calculator.js"></script>
   ```
4. Нажмите **«Сохранить»**.

> **Примечание:** файлы `css/` и `js/` нужно сначала разместить на вашем  
> хостинге (или использовать CDN — см. Часть 4).

---

### Метод B — Код на конкретной странице
Подходит для карты, калькулятора, виджета валют — только на нужных страницах.

1. Откройте редактор нужной страницы.
2. Нажмите **«Настройки страницы»** (шестерёнка ⚙️ наверху).
3. Перейдите на вкладку **«Дополнительно»** → **«HTML-код для секции HEAD»**.
4. Вставьте нужные теги `<link>` и `<script>`.

---

### Метод C — HTML-блок T123 (рекомендуется для карты и виджетов)
Позволяет вставить любой HTML/JS/CSS прямо в тело страницы.

1. В редакторе страницы нажмите **«+»** для добавления блока.
2. В поиске введите **«T123»** или выберите раздел **«Другое»** → **«HTML код»**.
3. Дважды кликните на блок, вставьте код. Пример для карты:
   ```html
   <div id="routeMapContainer" style="width:100%;height:600px;border-radius:16px;overflow:hidden;"></div>
   <script src="https://ВАШ_ДОМЕН/js/map.js"></script>
   ```
4. Нажмите **«Сохранить»** и **«Опубликовать страницу»**.

> ⚠️ Код в T123-блоке работает **только на опубликованной** странице,  
> в режиме редактора он отображается как серый блок-заглушка.

## Часть 3 — API Tilda (для программной интеграции)

Tilda предоставляет REST API для чтения и записи данных сайта.  
Требуется тарифный план **Business** или выше.

### Получение API-ключей:
1. **Настройки сайта** → **«Экспорт»** → **«API-интеграция»**.
2. Нажмите **«Сгенерировать ключи»** — получите:
   - `publickey` — можно передать разработчику.
   - `secretkey` — **держите в тайне**, не передавайте в браузерный код.
3. Пришлите ключи разработчику по защищённому каналу (не в открытом чате).

### Пример API-запроса (получить список страниц):
```
GET https://api.tildacdn.info/v1/getpageslist/
  ?publickey=ВАШ_PUBLICKEY
  &secretkey=ВАШ_SECRETKEY
  &projectid=ID_ВАШЕГО_САЙТА
```

### Что можно сделать через API:
| Метод | Описание |
|-------|----------|
| `getprojectslist` | Список ваших сайтов |
| `getpageslist` | Список страниц сайта |
| `getpage` | Полный HTML конкретной страницы |
| `getpagefull` | HTML + CSS + JS страницы |
| `exportproject` | Экспорт всего сайта |

## Часть 4 — Рекомендуемый способ размещения файлов JS/CSS

Поскольку Tilda не хранит произвольные статические файлы,  
разместите `css/` и `js/` папки одним из способов:

### Вариант A — GitHub Pages (бесплатно, рекомендуется)
1. Зайдите в репозиторий `bossssmann-ui/pacificstar.ru` на GitHub.
2. Перейдите в **Settings** → **Pages**.
3. В разделе **Source** выберите ветку `main` (или `copilot/refactor-site-description`), папку `/root`.
4. Нажмите **Save**. Через 1–2 минуты сайт будет доступен по адресу:
   ```
   https://bossssmann-ui.github.io/pacificstar.ru/
   ```
5. Используйте этот URL как базовый для всех ссылок на CSS/JS в Tilda.

### Вариант B — Ваш хостинг
Загрузите папки `css/` и `js/` на сервер через FTP/SFTP/панель хостинга  
и используйте URL вашего домена как базовый.

### Вариант C — jsDelivr CDN (для GitHub)
После публикации репозитория на GitHub файлы автоматически доступны через CDN:
```
https://cdn.jsdelivr.net/gh/bossssmann-ui/pacificstar.ru@main/css/style.css
https://cdn.jsdelivr.net/gh/bossssmann-ui/pacificstar.ru@main/js/main.js
```
Используйте эти URL в блоках Tilda — файлы загружаются с высокой скоростью по всему миру.

## Часть 5 — Интеграция AmoCRM через Tilda

После получения API-ключей AmoCRM:

1. В **Настройках сайта Tilda** → **«Аналитика»** → **«CRM»**.
2. Выберите **«AmoCRM»** из списка.
3. Введите субдомен вашего портала AmoCRM (например, `pacificstar.amocrm.ru`).
4. Нажмите **«Подключить»** — Tilda попросит авторизоваться в AmoCRM.
5. После авторизации все формы Tilda будут **автоматически создавать сделки**  
   в AmoCRM в указанную воронку.

Для форм на страницах, написанных в HTML (не Tilda-блоки),  
передайте разработчику API-ключи AmoCRM — он донастроит интеграцию в `js/main.js`.
