# 📧 Настройка SMTP-сервера для регистрации клиентов

> Пошаговая инструкция: куда заходить и что нажимать, чтобы при регистрации
> клиент получал письмо с подтверждением на свой e-mail.

---

## 🚀 Быстрый старт для `noreply@pacificstar.ru` (Яндекс)

> Ящик уже создан — осталось 3 действия.

### Прогресс:
- [x] **Шаг 1.** Создать ящик `noreply@pacificstar.ru` — ✅ Готово
- [ ] **Шаг 2.** Получить пароль приложения в Яндексе — 👈 **ВЫ ЗДЕСЬ**
- [ ] **Шаг 3.** Создать файл `.env` на сервере
- [ ] **Шаг 4.** Запустить сервер и проверить

### 👉 Ваш следующий шаг — получить пароль приложения:

**1.** Откройте в браузере (войдите под аккаунтом, к которому привязан `noreply@pacificstar.ru`):

👉 https://id.yandex.ru/security/app-passwords

**2.** Яндекс попросит включить **двухфакторную аутентификацию** (если ещё не включена):
   - Нажмите **«Включить»**
   - Скачайте приложение **Яндекс Ключ** на телефон (App Store / Google Play)
   - Отсканируйте QR-код с экрана
   - Введите код из приложения

**3.** После включения 2FA — вернитесь на ту же страницу:

👉 https://id.yandex.ru/security/app-passwords

**4.** Нажмите кнопку **«Создать пароль приложения»**

**5.** В поле «Название» напишите: `Pacific Star сайт`

**6.** Тип выберите: **«Почта»**

**7.** Нажмите **«Создать»**

**8.** На экране появится пароль — **скопируйте его и сохраните!**

> ⚠️ Этот пароль показывается **один раз**. Если потеряете — придётся создавать новый.

**Когда пароль скопирован** — переходите к **[Шагу 3](#шаг-3-создать-файл-env-на-сервере)**, где нужно вставить его в файл `.env`:

```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=noreply@pacificstar.ru
SMTP_PASS=вставьте-скопированный-пароль-сюда

PORT=3000

MAIL_FROM="Pacific Star <noreply@pacificstar.ru>"
```

---

## Как это работает

1. Клиент заполняет форму на странице **Личный кабинет** (`account.html`)
2. Сервер (`server.js`) получает данные и отправляет письмо через SMTP
3. Клиент видит подтверждение в почте

Без SMTP письма **не уходят** — сервер просто пишет их в консоль (режим разработки).

## Шаг 1. Создать почтовый ящик для отправки

> Можно использовать существующий ящик, но лучше завести отдельный — например,
> `noreply@pacificstar.ru` или `info@pacificstar.ru`.

### Вариант A: Яндекс (рекомендуется)

**1.** Откройте в браузере:
👉 https://mail.yandex.ru/

**2.** Войдите в аккаунт (или создайте новый)

**3.** Если у вас свой домен (`pacificstar.ru`) — подключите его:
- Перейдите в **Яндекс 360 для бизнеса**: https://admin.yandex.ru/
- Нажмите **«Подключить домен»**
- Введите `pacificstar.ru`
- Следуйте инструкции (добавьте DNS-записи)
- Создайте ящик `noreply@pacificstar.ru`

**4.** Если свой домен не нужен — используйте обычный ящик `@yandex.ru`

### Вариант B: Mail.ru

**1.** Откройте: 👉 https://mail.ru/

**2.** Создайте ящик или используйте существующий

**3.** Для своего домена — Biz.Mail.ru: https://biz.mail.ru/

### Вариант C: Gmail

**1.** Откройте: 👉 https://mail.google.com/

**2.** Создайте аккаунт или используйте существующий

> ⚠️ Gmail часто блокирует отправку с серверов. Яндекс и Mail.ru работают надёжнее.

## Шаг 2. Получить пароль приложения

> **Обычный пароль от почты НЕ подойдёт** — нужен специальный «пароль приложения».
> Это требование безопасности почтовых провайдеров.

### Для Яндекса

**1.** Откройте в браузере:
👉 https://id.yandex.ru/security/app-passwords

**2.** Если появится запрос — включите **двухфакторную аутентификацию**:
- Нажмите **«Включить»**
- Отсканируйте QR-код приложением **Яндекс Ключ** (скачайте из App Store / Google Play)
- Введите код подтверждения

**3.** После включения 2FA вернитесь на страницу паролей приложений:
👉 https://id.yandex.ru/security/app-passwords

**4.** Нажмите **«Создать пароль приложения»**

**5.** В поле «Название» введите: `Pacific Star сайт`

**6.** Выберите тип: **«Почта»**

**7.** Нажмите **«Создать»**

**8.** **Скопируйте пароль** (он покажется один раз!) — сохраните его в надёжном месте

### Для Mail.ru

**1.** Откройте: 👉 https://account.mail.ru/user/2-step-auth/passwords

**2.** Нажмите **«Добавить»**

**3.** Введите название: `Pacific Star`

**4.** Нажмите **«Создать»**

**5.** Скопируйте и сохраните пароль

### Для Gmail

**1.** Откройте: 👉 https://myaccount.google.com/apppasswords

**2.** Введите название: `Pacific Star`

**3.** Нажмите **«Создать»**

**4.** Скопируйте 16-символьный пароль

## Шаг 3. Создать файл `.env` на сервере

**1.** Подключитесь к серверу по SSH (или откройте файловый менеджер в панели хостинга)

**2.** Перейдите в папку проекта — туда, где лежит файл `server.js`:
```bash
cd /home/ваш_логин/pacificstar.ru
```

**3.** Создайте файл `.env`:
```bash
nano .env
```

**4.** Вставьте следующее содержимое (замените значения на свои):

### Для Яндекса:
```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=noreply@pacificstar.ru
SMTP_PASS=вставьте-пароль-приложения-из-шага-2

PORT=3000

MAIL_FROM="Pacific Star <noreply@pacificstar.ru>"
```

### Для Mail.ru:
```env
SMTP_HOST=smtp.mail.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your-email@mail.ru
SMTP_PASS=вставьте-пароль-приложения-из-шага-2

PORT=3000

MAIL_FROM="Pacific Star <your-email@mail.ru>"
```

### Для Gmail:
```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your-email@gmail.com
SMTP_PASS=вставьте-пароль-приложения-из-шага-2

PORT=3000

MAIL_FROM="Pacific Star <your-email@gmail.com>"
```

**5.** Сохраните файл:
- Нажмите **`Ctrl + O`** → **`Enter`** (сохранить)
- Нажмите **`Ctrl + X`** (выйти)

## Шаг 4. Установить зависимости и запустить сервер

**1.** Убедитесь, что Node.js установлен:
```bash
node -v
```
Должна быть версия 18 или выше. Если нет — установите:
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**2.** Установите зависимости проекта:
```bash
npm install
```

**3.** Запустите сервер:
```bash
npm start
```

**4.** Если всё настроено правильно, в консоли вы увидите:
```
🚀 Pacific Star server running on http://localhost:3000
✅ SMTP connected: smtp.yandex.ru
```

Если SMTP **не настроен** или данные неверные, будет:
```
⚠️  SMTP not configured — emails will be logged to console
```
или
```
❌ SMTP verify failed: ...сообщение об ошибке...
```

## Шаг 5. Проверить отправку писем

**1.** Откройте в браузере:
👉 `http://ваш-сервер:3000/account`

**2.** Перейдите на вкладку **«Регистрация»**

**3.** Заполните форму:
- **Имя**: Тест
- **Фамилия**: Тестов
- **E-mail**: ваш-реальный-адрес@почта.ru (куда придёт письмо)
- **Телефон**: любой
- **Пароль**: любой (8+ символов)

**4.** Нажмите **«Зарегистрироваться»**

**5.** Проверьте указанный e-mail — должно прийти письмо **«Регистрация на Pacific Star — подтверждение»**

**6.** Также проверьте API напрямую — откройте в браузере:
👉 `http://ваш-сервер:3000/api/health`

В ответе должно быть `"smtp": true`:
```json
{
  "ok": true,
  "smtp": true,
  "time": "2026-03-06T10:00:00.000Z"
}
```

## Шаг 6 (опционально). Запуск сервера в фоне

Чтобы сервер работал постоянно (не выключался при закрытии SSH), используйте **pm2**:

**1.** Установите pm2:
```bash
npm install -g pm2
```

**2.** Запустите сервер через pm2:
```bash
pm2 start server.js --name pacificstar
```

**3.** Настройте автозапуск при перезагрузке:
```bash
pm2 startup
pm2 save
```

**4.** Полезные команды pm2:
```bash
pm2 status            # Статус всех процессов
pm2 logs pacificstar  # Логи сервера
pm2 restart pacificstar  # Перезапуск
pm2 stop pacificstar     # Остановка
```

## ❓ Частые проблемы и решения

### «SMTP verify failed: Invalid login»
- Проверьте `SMTP_USER` — должен быть полный e-mail (например `noreply@pacificstar.ru`)
- Проверьте `SMTP_PASS` — это **пароль приложения**, а не обычный пароль
- Убедитесь, что 2FA включена (для Яндекса это обязательно)

### «SMTP not configured — emails will be logged to console»
- Файл `.env` не найден — проверьте, что он лежит **рядом с `server.js`**
- Одна из переменных `SMTP_HOST`, `SMTP_USER` или `SMTP_PASS` пустая
- После изменения `.env` нужно **перезапустить** сервер

### Письмо не приходит (но в логах `Mail sent`)
- Проверьте папку **«Спам»** у получателя
- Убедитесь, что домен отправителя настроен правильно (SPF, DKIM записи)
- Попробуйте отправить на другой ящик

### «Connection timeout» / «Connection refused»
- Проверьте `SMTP_PORT` — для Яндекса и Mail.ru это `465`
- Убедитесь, что хостинг **не блокирует** исходящие соединения на порт 465
- Попробуйте порт `587` и `SMTP_SECURE=false` (STARTTLS):
```env
SMTP_PORT=587
SMTP_SECURE=false
```

### На dashboard появляется «Не удалось отправить подтверждение»
- Это значит, что сервер ответил ошибкой при отправке письма
- Проверьте логи сервера: `pm2 logs pacificstar` или консоль, где запущен `npm start`
- Исправьте настройки `.env` и перезапустите сервер

## 📋 Сводка всех переменных `.env`

| Переменная | Описание | Пример |
|------------|----------|--------|
| `SMTP_HOST` | Адрес SMTP-сервера | `smtp.yandex.ru` |
| `SMTP_PORT` | Порт SMTP | `465` |
| `SMTP_SECURE` | Использовать SSL | `true` |
| `SMTP_USER` | Логин (полный e-mail) | `noreply@pacificstar.ru` |
| `SMTP_PASS` | Пароль приложения | `abcdefghijklmnop` |
| `PORT` | Порт веб-сервера | `3000` |
| `MAIL_FROM` | Имя и адрес отправителя | `"Pacific Star <noreply@pacificstar.ru>"` |

## 🔒 Про безопасность

- Файл `.env` **никогда не должен попадать в Git** — он уже добавлен в `.gitignore`
- Используйте **пароль приложения**, а не основной пароль от почты
- Рекомендуется завести **отдельный** почтовый ящик только для отправки с сайта
- Если пароль скомпрометирован — отзовите его в настройках почты и создайте новый
