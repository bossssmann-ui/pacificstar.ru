# 🚀 Запуск сайта на Timeweb — одно действие

> Всё что нужно сделать вам — **один раз добавить пароль в GitHub**.  
> После этого я буду обновлять сайт автоматически.

---

## ✅ Данные сервера

| Что | Значение |
|-----|----------|
| Адрес сервера | `ваш_IP` (GitHub Secret: `TIMEWEB_HOST`) |
| Логин | `ваш_логин` (GitHub Secret: `TIMEWEB_USER`) |
| Папка сайта | `/home/ваш_логин/public_html/` |
| Пароль | ← нужно добавить один раз (Шаг ниже) |

---

## 🔐 Единственное действие: добавить секреты в GitHub

> Зачем это нужно: GitHub должен знать данные сервера, чтобы загружать файлы.  
> Данные хранятся зашифровано — никто, кроме автоматики, их не увидит.

### Добавьте три секрета:

**1.** Откройте эту ссылку в браузере:  
👉 https://github.com/bossssmann-ui/pacificstar.ru/settings/secrets/actions

**2.** Нажмите зелёную кнопку **`New repository secret`** и добавьте каждый из трёх секретов:

| Name | Secret |
|------|--------|
| `TIMEWEB_HOST` | IP-адрес сервера (из панели Timeweb) |
| `TIMEWEB_USER` | Логин на хостинге |
| `TIMEWEB_PASSWORD` | Пароль от хостинга |

**3.** Нажмите **`Add secret`** для каждого.

---

## ▶️ Запустить загрузку сайта

После добавления пароля нужно запустить загрузку вручную **один раз**:

**1.** Откройте:  
👉 https://github.com/bossssmann-ui/pacificstar.ru/actions/workflows/tilda-deploy.yml

**2.** Нажмите серую кнопку **`Run workflow`** (справа)

**3.** Ещё раз нажмите зелёную кнопку **`Run workflow`**

**4.** Подождите 1–2 минуты — увидите зелёную галочку ✅

**5.** Откройте **pacificstar.ru** — сайт работает! 🎉

## 🔒 Про безопасность

- Пароль в GitHub Secrets **зашифрован** — даже я не вижу его в открытом виде
- В коде сайта пароль **нигде не написан**
- Загрузка идёт через зашифрованный канал (SSH)
- Рекомендую после первого успешного деплоя **сменить пароль** на Timeweb  
  (и обновить в GitHub Secrets) — это хорошая практика безопасности

## 🔧 Backend (формы) — Фаза 0

Статика деплоится через GitHub Actions (см. выше). **Формы** (`/api/contact`, обратный звонок) требуют Node.js.

На **shared hosting** (`public_html`) постоянный Node.js **недоступен** — используйте **Timeweb Cloud App Platform** (см. ниже).

### App Platform (рекомендуется)

Приложение уже создано: [дашборд app #220769](https://timeweb.cloud/my/apps/220769/dashboard).

| Параметр | Значение |
|----------|----------|
| Тип | Backend → Express (или Dockerfile — см. ниже) |
| Репозиторий | `bossssmann-ui/pacificstar.ru`, ветка `main` |
| Команда сборки | `npm ci --omit=dev` |
| Команда запуска | **`npm start`** (важно — не оставлять PM2 по умолчанию, иначе ENV из панели может не подхватиться) |
| Health check | `/api/health` |

> **Не задавайте `PORT` вручную.** Платформа сама передаёт `PORT`; если прописать `PORT=3000`, nginx не достучится до приложения — домен не откроется.

**Переменные окружения** — только через **Настройки → Настройка деплоя → Редактировать → Переменные** (не вкладка «Переменные» в меню App Platform слева, если переменные там не привязаны к приложению).

> **Важно:** в Timeweb нельзя вставить весь блок `KEY=VALUE` в одно поле. Каждая переменная — отдельная строка: кнопка **«Добавить»** → поле **Имя** + поле **Значение** → **Сохранить** внизу окна. После сохранения — **новый деплой** (вкладка «Деплой») или **Перезагрузка приложения**.

| Имя (поле «Имя») | Значение (поле «Значение») |
|-----|----------|
| `SMTP_HOST` | `smtp.yandex.ru` |
| `SMTP_PORT` | `465` |
| `SMTP_SECURE` | `true` |
| `SMTP_USER` | `noreply@pacificstar.ru` |
| `SMTP_PASS` | **пароль приложения Яндекс** (только пароль, без кавычек) |
| `MAIL_FROM` | `noreply@pacificstar.ru` *(сначала без «Pacific Star <…>» — проще)* |
| `CORS_ORIGIN` | `https://pacificstar.ru` |
| `API_ONLY` | `true` |
| `CONTACT_EMAIL` | `sales@pacificstar.ru` |

**Не добавляйте** `PORT` — платформа задаёт его автоматически.

Опционально: `AMOCRM_WEBHOOK_URL`, `PS_YM_ID` (для аналитики — в `js/config.js` на статике).

#### Домен не открывается (`bossssmann-ui-pacificstar-ru-73ae.twc1.net`)

Симптом: браузер «не удаётся установить соединение» / TLS error. DNS есть, но приложение не отвечает.

**Пошагово в панели App Platform:**

1. **Настройки → Переменные** — удалите `PORT`, если она есть.
2. **Настройки → Настройки деплоя → Редактировать:**
   - Сборка: `npm ci --omit=dev`
   - Запуск: **`npm start`** (обязательно — PM2 по умолчанию кэширует старое окружение без SMTP)
   - Health check: `/api/health`
3. **Деплой** → «Выполнить откат к коммиту» / перезапуск последнего коммита.
4. **Логи приложения** (не только лог деплоя!) — ищите:
   - `Pacific Star boot: PORT=...`
   - `🚀 Pacific Star server listening on 0.0.0.0:...`
   - ошибки `Error:`, `Cannot find module`
5. Если Express не заводится — **пересоздайте приложение через Dockerfile** (см. ниже).

**Текущий URL API:** `https://bossssmann-ui-pacificstar-ru-73ae.twc1.net`

#### ERR_CONNECTION_TIMED_OUT / таймаут

Симптом: `bossssmann-ui-pacificstar-ru-73ae.twc1.net` не открывается, формы на сайте падают.

**Срочно в панели:**

1. [Дашборд #220769](https://timeweb.cloud/my/apps/220769/dashboard) → статус **«Запущено»** (не «Остановлено»).
2. **Переменные** → удалить `PORT`, если есть.
3. **Перезагрузка приложения** (кнопка restart).
4. **Логи приложения** → последние 30 строк (скриншот, если не помогло).

Если после restart всё ещё таймаут — **пересоздайте backend через Dockerfile** (надёжнее Express-шаблона):

| Шаг | Действие |
|-----|----------|
| 1 | App Platform → **Добавить** → тип **Dockerfile** |
| 2 | Репо `bossssmann-ui/pacificstar.ru`, ветка `main` |
| 3 | Путь к Dockerfile: корень репо (файл `Dockerfile` уже в репо) |
| 4 | Health check: `/api/health` |
| 5 | ENV: `API_ONLY=true`, `CORS_ORIGIN`, `SMTP_*`, `CONTACT_EMAIL=sales@pacificstar.ru` |
| 6 | **Не добавлять** `PORT` вручную |
| 7 | После деплоя — новый `*.twc1.net` на вкладке «Дашборд» → прислать нам одной строкой |

Старый URL `bossssmann-ui-pacificstar-ru-73ae.twc1.net` можно удалить после переключения.

#### Было работало — снова не открывается

Часто после добавления SMTP или перезапуска:

1. **Дашборд** — статус приложения «Запущено» (не «Остановлено»).
2. **Переменные** — снова проверьте: **нет `PORT`**, есть `API_ONLY=true`.
3. **Настройки → Перезагрузка приложения** (кнопка перезапуска контейнера).
4. **Деплой** — последний деплой зелёный? Если красный — откройте лог.
5. **Логи приложения** — последние строки (скриншот нам, если неясно).

После смены переменных Timeweb иногда нужен **явный перезапуск**, не только сохранение ENV.

#### Логи: «SMTP not configured» / `SMTP_USER=(empty)`

Типичный вывод (переменные **не дошли** до контейнера):

```
⚠️  SMTP not configured — emails will be logged to console
Pacific Star boot: PORT=8080 ... SMTP_USER=(empty) MAIL_FROM=(empty) API_ONLY=false (raw=unset)
⚠️  App Platform ENV missing (9): API_ONLY, CONTACT_EMAIL, ...
```

| Строка в логе | Что значит |
|---------------|------------|
| `SMTP_USER=(empty)` | `SMTP_*` не заданы в **локальных** переменных приложения |
| `API_ONLY=false (raw=unset)` | `API_ONLY=true` не задан — backend отдаёт и статику (лишнее на App Platform) |
| `CONTACT_EMAIL=... (env=default)` | адрес из кода, не из панели |
| `injected env (0) from .env` | файла `.env` в контейнере нет — **это нормально**; переменные должны приходить из панели |

**Что сделать (5 минут):**

1. [App #220769](https://timeweb.cloud/my/apps/220769/dashboard) → **Настройки** → **Настройка деплоя** → **Редактировать**.
2. Блок **Переменные** → для **каждой** строки из таблицы выше: **Добавить** → Имя / Значение (не вставляйте `API_ONLY=true` целиком в одно поле).
3. Внизу окна — **Сохранить**. На вкладке «Настройки» в списке переменных должно быть **9 штук**.
4. **Деплой** → запустить деплой заново **или** **Перезагрузка приложения** (одного «Сохранить» мало).
5. **Команда запуска** в настройках деплоя должна быть **`npm start`**, не пустая (PM2).
6. Проверка:

**Альтернатива — загрузка файла:** создайте на компьютере файл с именем ровно `.env` (без `.txt`), без строк с `#`, вставьте содержимое из [`.env.app-platform.example`](.env.app-platform.example), подставьте `SMTP_PASS` → **Загрузить из файла**.

```bash
curl -s https://bossssmann-ui-pacificstar-ru-73ae.twc1.net/api/health
```

Ожидается: `"smtp":true`, `"apiOnly":true`, в `env` все поля `true`.

> Глобальные переменные (App Platform → Переменные) **не подхватятся**, пока вы не привяжете их к приложению в «Настройка деплоя».

#### Где взять публичный URL API

Ссылка на дашборд (`…/apps/220769/dashboard`) — **не** адрес API.

1. Откройте [дашборд приложения](https://timeweb.cloud/my/apps/220769/dashboard).
2. На вкладке **«Дашборд»** найдите **технический домен** — формат `https://что-то.twc1.net`.
3. Проверка в терминале:

```bash
curl -s https://ВАШ-ДОМЕН.twc1.net/api/health
# ожидается: {"ok":true,"smtp":true,...}
```

4. **Пришлите одной строкой** этот URL — агент пропишет его в `js/config.js` (`PS_API_BASE`) и задеплоит статику.

После подключения формы на **pacificstar.ru** будут слать запросы на App Platform (CORS уже настроен в `server.js`).

#### Опционально: свой домен `api.pacificstar.ru`

Настройки → Домены → добавить домен → A-запись на IP приложения (виден на дашборде). Тогда `PS_API_BASE = 'https://api.pacificstar.ru'`.

#### Безопасность

- Удалите `.env` из `public_html` на shared hosting — секреты только в App Platform.
- `server.js` в `public_html` можно оставить или убрать (сейчас отдаётся как статический файл, не выполняется).
