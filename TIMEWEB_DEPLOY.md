# 🚀 Запуск сайта на Timeweb — одно действие

> Всё что нужно сделать вам — **один раз добавить пароль в GitHub**.  
> После этого я буду обновлять сайт автоматически.

---

## ✅ Данные сервера

| Что | Значение |
|-----|----------|
| Адрес сервера | `ваш_IP` (GitHub Secret: `TIMEWEB_HOST`) |
| Логин | `ваш_логин` (GitHub Secret: `TIMEWEB_USER`) |
| Папка сайта | `/home/ваш_логин/public_html/` |
| Пароль | ← нужно добавить один раз (Шаг ниже) |

---

## 🔐 Единственное действие: добавить секреты в GitHub

> Зачем это нужно: GitHub должен знать данные сервера, чтобы загружать файлы.  
> Данные хранятся зашифрованно — никто, кроме автоматики, их не увидит.

### Добавьте три секрета:

**1.** Откройте эту ссылку в браузере:  
👉 https://github.com/bossssmann-ui/pacificstar.ru/settings/secrets/actions

**2.** Нажмите зелёную кнопку **`New repository secret`** и добавьте каждый из трёх секретов:

| Name | Secret |
|------|--------|
| `TIMEWEB_HOST` | IP-адрес сервера (из панели Timeweb) |
| `TIMEWEB_USER` | Логин на хостинге |
| `TIMEWEB_PASSWORD` | Пароль от хостинга |

**3.** Нажмите **`Add secret`** для каждого.

---

## ▶️ Запустить загрузку сайта

После добавления пароля нужно запустить загрузку вручную **один раз**:

**1.** Откройте:  
👉 https://github.com/bossssmann-ui/pacificstar.ru/actions/workflows/tilda-deploy.yml

**2.** Нажмите серую кнопку **`Run workflow`** (справа)

**3.** Ещё раз нажмите зелёную кнопку **`Run workflow`**

**4.** Подождите 1–2 минуты — увидите зелёную галочку ✅

**5.** Откройте **pacificstar.ru** — сайт работает! 🎉

## ❓ Что делать если что-то не так

**Увидели красный крест ❌ вместо зелёной галочки?**  
Сделайте скриншот и пришлите мне — разберёмся вместе.

**Сайт не открывается после деплоя?**  
Возможно, папка на сервере другая. Напишите мне — проверю и исправлю.

**Хотите сменить пароль на Timeweb?**  
Смените — потом просто обновите значение `TIMEWEB_PASSWORD` в GitHub Secrets по той же инструкции.

---

## 🔒 Про безопасность

- Пароль в GitHub Secrets **зашифрован** — даже я не вижу его в открытом виде
- В коде сайта пароль **нигде не написан**
- Загрузка идёт через зашифрованный канал (SSH)
- Рекомендую после первого успешного деплоя **сменить пароль** на Timeweb  
  (и обновить в GitHub Secrets) — это хорошая практика безопасности
