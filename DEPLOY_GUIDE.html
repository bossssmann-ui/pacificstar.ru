Пошаговая инструкция — картинка за картинкой

Что нужно сделать: Всего 2 части — один раз добавить пароль, один раз нажать кнопку. Займёт 5 минут. После этого сайт будет обновляться автоматически при любых изменениях.
1

Открыть страницу с секретами GitHub

Нажмите на ссылку ниже — откроется нужная страница в вашем браузере:

https://github.com/bossssmann-ui/pacificstar.ru/settings/secrets/actions

Если попросит войти — войдите под своим аккаунтом GitHub (bossssmann-ui).

Вы увидите примерно такую страницу:

👆
Нажмите зелёную кнопку «+ New repository secret»
2

Заполнить форму (2 поля)

После клика на кнопку откроется форма. Заполните два поля:

github.com/bossssmann-ui/pacificstar.ru/settings/secrets/actions/new
New secret
Напишите точно так, буква в букву, всё заглавными
Сюда вводите ваш пароль от Timeweb (тот, которым входите в панель управления)
Add secret
Поле Name: введите ровно TIMEWEB_PASSWORD — заглавными, без пробелов

Поле Secret: введите ваш пароль от Timeweb

Затем нажмите зелёную кнопку «Add secret»
3

Убедиться что пароль добавлен

После нажатия «Add secret» вернётесь на предыдущую страницу. Теперь она будет выглядеть так:

github.com/bossssmann-ui/pacificstar.ru/settings/secrets/actions

TIMEWEB_PASSWORD
Updated just now

✅

Отлично! Пароль добавлен.
Теперь переходим к запуску.

4

Открыть страницу запуска

Нажмите на ссылку:

https://github.com/bossssmann-ui/pacificstar.ru/actions/workflows/tilda-deploy.yml
⚠️ ВНИМАНИЕ — ДВЕ РАЗНЫЕ КНОПКИ НА ОДНОЙ СТРАНИЦЕ!

На этой странице есть строка-фильтр слева и кнопка запуска справа. Они похожи, но делают разное:

❌ НЕ ЭТА
Строка фильтра (слева)
Если нажать — откроется меню с пунктами:
«actor», «branch», «event», «is»

Это фильтр для поиска по истории запусков. Нам не нужен.
✅ ВОТ ЭТА
Кнопка «Run workflow» (справа)
Отдельная кнопка, она находится справа сверху над списком запусков.

Если нажать — появится маленькое меню с зелёной кнопкой внутри.

Вот как выглядит страница — нужная кнопка обведена зелёным:

github.com/bossssmann-ui/pacificstar.ru/actions/workflows/tilda-deploy.yml
Workflows
Deploy — pacificstar.ru на Timeweb
🔍 Filter workflow runs
❌ НЕ СЮДА
(это фильтр — открывает actor/branch/event/is)
✅ НАЖАТЬ СЮДА!
(список запусков пока пуст — это нормально)
Если кнопки «Run workflow» не видно:
Возможно страница показывает историю в виде таблицы — прокрутите немного вверх, кнопка должна быть в верхней правой части страницы над таблицей.
5

Подтвердить запуск в выпавшем меню

После клика появится маленькое меню — нажмите зелёную кнопку внутри него:

Use workflow from
👆
Нажмите зелёную кнопку «Run workflow» внутри меню
Ветку менять не нужно — оставьте как есть. Просто нажмите зелёную кнопку.
6

Подождать 1–2 минуты

После нажатия страница обновится. Появится строчка с жёлтым кружком — это значит «идёт загрузка»:

Как прислать мне скриншот?

Если что-то пошло не так — пришлите скриншот вот так:

  • На клавиатуре нажмите Print Screen (или Win+Shift+S на Windows, Cmd+Shift+4 на Mac) — выделите нужную область экрана
  • Откройте страницу нашего проекта: github.com/bossssmann-ui/pacificstar.ru/pulls
  • Нажмите на открытый Pull Request (Pull request №1)
  • Прокрутите вниз до поля «Leave a comment»
  • Вставьте скриншот в поле комментария Ctrl+V (или перетащите файл мышью)
  • Нажмите зелёную кнопку «Comment»
Проще: Просто опишите что вы видите — «красный крест», «кнопки нет», «страница пустая» — и я разберусь. Скриншот необязателен.
