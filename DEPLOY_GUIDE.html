🚢 Pacific Star — Запуск сайта

Инструкция по замене старого сайта Tilda на новый сайт Pacific Star

⚠️ Почему показывается белый экран и логотип Tilda?

Timeweb при распаковке ZIP не перезаписывает файлы, которые уже существуют. Старый index.html от Tilda остался в public_html/ — именно его и открывает браузер.

Наш index.html из ZIP был проигнорирован.

✅ Способ 1 — go.php (рекомендуется, 3 шага)

1

Скачайте go.php на компьютер

⬇ Скачать go.php

💡 Если вы уже распаковывали site-pacificstar.zip на Timeweb — go.php уже в папке public_html.
Пропустите шаги 1 и 2, сразу перейдите к шагу 3.

2

Загрузите go.php на Timeweb

Точно так же, как вы загружали hello.html:

Файловый менеджер Timeweb
📁 public_html ← вы здесь
  📄 index.html  ← старый Tilda (не трогайте)
  📄 hello.html  ← этот работает
  📄 about.html
  📁 css/
  📁 js/
  📄 go.php  ← загрузите сюда
3

Откройте go.php в браузере

▶ Открыть pacificstar.ru/go.php

Через 20–30 секунд браузер автоматически откроет pacificstar.ru с синей шапкой. go.php самоудалится — убирать вручную не нужно.

Если открылась синяя шапка = сайт работает! ✅
Если белый экран / ошибки: прислать скриншот в PR на GitHub

🔧 Способ 2 — вручную в файловом менеджере (если go.php не работает)

ШагДействие
1Timeweb → Файловый менеджер → public_html
2Найти старый index.html (от Tilda) → удалить его
3Найти наш index.html из site-pacificstar/ → переместить в корень
4То же с .htaccess — удалить старый Tilda, оставить наш
5Открыть pacificstar.ru — синяя шапка ✅

🚀 Авто-деплой (после первого запуска сайта)

Чтобы Copilot автоматически обновлял сайт при каждом изменении — добавьте пароль в GitHub один раз:

ШагДействие
1Открыть: GitHub Secrets
2«New repository secret» → Name: TIMEWEB_PASSWORD → вставить пароль → Save
3Запуск деплоя: Actions → Run workflow
