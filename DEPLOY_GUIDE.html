Два способа — выберите любой

⚠️ Почему виден логотип Тильды, а не наш сайт

На сервере сейчас лежат два index.html:

❌ СЕЙЧАС (неправильно)
📁 public_html/
  📄 index.html ← СТАРЫЙ (Tilda)
  📁 site-pacificstar/
    📄 index.html ← НАШ
    📁 css/
    📁 js/
✅ НАДО (правильно)
📁 public_html/
  📄 index.html ← НАШ
  📁 css/
  📁 js/
  
  (папка site-pacificstar удалена)

Сервер открывает первый index.html который находит в корне — это старый Tilda. Наш сайт никто не видит, он спрятан в подпапке.

Способ 1: Файловый менеджер РЕКОМЕНДУЮ

Перемещаем наши файлы из подпапки в корень. Быстро и надёжно.

  1. Войдите в Timeweb → Файловый менеджер
  2. Откройте папку public_html
  3. Найдите и откройте папку site-pacificstar
  4. Нажмите галочку в шапке — выделить все файлы
  5. Нажмите кнопку «Переместить» (или правой кнопкой → Вырезать)
  6. Нажмите «Назад» — вернитесь в public_html
  7. Нажмите «Вставить» — подтвердите перезапись index.html
  8. Удалите теперь пустую папку site-pacificstar

✅ Шаг 7 заменит старый Tilda index.html нашим сайтом

Открыть Timeweb →

Способ 2: Автоматический деплой

GitHub сам загружает все файлы на сервер. Нажать 2 кнопки.

  1. Откройте: GitHub Actions → Deploy
  2. Нажмите серую кнопку «Run workflow» (справа вверху)
  3. В выпавшем меню нажмите зелёную «Run workflow»
  4. Подождите 2 минуты → зелёная галочка ✅
  5. Откройте pacificstar.ru

Требует: пароль TIMEWEB_PASSWORD добавлен в Secrets

Открыть GitHub Actions →

✅ Как понять что сайт работает

Откройте pacificstar.ru — должна появиться тёмно-синяя шапка с логотипом «Pacific Star» и меню сверху.

Видите синюю шапку — готово! Пишите мне, продолжим.

Видите белый экран + Тильда — напишите мне что видно в файловом менеджере, я подскажу.
