⬜ Пустая страница?

Понятно что случилось — и есть простое решение

Когда вы распаковали ZIP, файловый менеджер Timeweb автоматически создал папку «site-pacificstar» внутри public_html. Получилось вот так:

📁 public_html/
📁 site-pacificstar/  ← лишняя папка!
📄 index.html  ← сайт спрятан здесь

Сервер ищет index.html прямо в public_html, но находит только пустую папку. Отсюда пустая страница.

Решение: загрузите скрипт fix.php — он сам переместит все файлы на нужное место за секунду.

🔧 Исправляем за 3 шага

Займёт около 2 минут

1
Скачайте файл-помощник fix.php
Нажмите кнопку — файл сохранится в папку «Загрузки»
⬇️ Скачать fix.php
Это безопасный файл — он только перемещает файлы сайта на нужное место. После работы его нужно удалить (напомним об этом).
2
Загрузите fix.php на Timeweb
В ту же папку public_html
🌐 Открыть Timeweb
1️⃣ Зайдите в Файловый менеджер → папка public_html
2️⃣ Нажмите «Загрузить файл»
3️⃣ Выберите fix.php из папки «Загрузки»
4️⃣ Дождитесь загрузки (маленький файл — секунда)
📁 public_html/  ← вы здесь
📁 site-pacificstar/  ← папка с сайтом
🔧 fix.php ← только что загрузили
3
Откройте fix.php в браузере
Он сам сделает всё за вас — за секунду
▶️ Открыть pacificstar.ru/fix.php

Что должны увидеть — зелёный экран:

🎉
Готово! Сайт установлен
Все файлы перемещены на нужное место
🌐 Открыть сайт →
↑ нажмите эту кнопку
После того как сайт откроется — вернитесь в Файловый менеджер и удалите fix.php (правая кнопка → Удалить). Он больше не нужен.
✅

Готово!

Напишите разработчику: «Сайт работает» — и перейдём к следующим улучшениям.

🌐 Открыть pacificstar.ru

⚠️ Если fix.php написал «Не удалось найти файлы»

Значит файловый менеджер сам уже переместил файлы, или структура другая. Попробуйте открыть pacificstar.ru — возможно, сайт уже работает!
Если нет — напишите разработчику что написано на экране fix.php.
