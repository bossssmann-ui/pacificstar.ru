Интернет-анализ

Варианты карт для согласования

Собрали 4 реалистичных сценария для Pacific Star: от локальной SVG-карты без API до полноценных платформ с поиском, геокодированием и маршрутизацией.

🧭

Локальная SVG-карта

Без внешних API

✅ Уже реализовано

Подходит для презентации маршрутов, хабов и торговых направлений. Максимально быстрая загрузка, полный контроль над стилем и отсутствие зависимости от сторонних сервисов.

Лучше всего: имиджевая карта на главной, фиксированные маршруты, работа без API-ключей.

Ограничение: нет поиска адресов, пробок и построения маршрута до точки клиента.

Источник: текущая реализация Pacific Star

🗺️

Leaflet + OpenStreetMap

Open source стек

⚖️ Гибко и недорого

Интерактивная карта с маркерами, слоями и pop-up подсказками. Подходит, если нужна свобода кастомизации без привязки к одному вендору.

Лучше всего: офисы, склады, партнёрские точки, фильтры по регионам и простые маршруты.

Ограничение: для заметного трафика лучше использовать свой или коммерческий тайл-сервис, а не публичные OSM tiles.

Источники: Leaflet Docs OSM Tile Policy

📍

Яндекс Карты JS API

Лучший вариант для РФ

⭐ Рекомендуем проверить первым

Самый практичный вариант для российского бизнеса: точная адресная база, маршруты, геокодирование и привычный интерфейс для пользователей.

Лучше всего: контакты, офисы, склады, поиск адреса и построение маршрута по России.

Ограничение: коммерческая лицензия и менее выразительная международная карта для продуктовой витрины.

Источники: Yandex Maps API Коммерческие условия

🚚

Mapbox GL JS

Премиальный UX

🎨 Лучшая визуализация

Векторная карта с красивыми стилями, анимацией, кластеризацией и богатой работой с live-данными. Хороша для трекинга и product-style интерфейсов.

Лучше всего: живой трекинг, брендовые стили, сложные слои и клиентские dashboards.

Ограничение: зарубежный биллинг и pay-as-you-go модель — нужно заранее считать стоимость и риски доступности.

Источники: Mapbox Docs Pricing

Предварительная рекомендация: оставить SVG-карту как имиджевую карту маршрутов на главной, а для прикладных сценариев (офис, склад, построение маршрута клиента) отдельно протестировать Яндекс Карты JS API.

Дорожная карта

План развития 2025–2026

Приоритизированный список интеграций с оценкой пользы для клиентов и команды

Q2
2025

AmoCRM: полная интеграция форм

Все формы сайта (заявка, обратный звонок, регистрация) создают сделки в AmoCRM с тегами источника, суммой и маршрутом. Менеджер видит лид за секунды. Польза для команды: нет ручного переноса заявок. Польза для клиента: быстрый отклик.

Q2
2025

Яндекс.Метрика + цели конверсии

Счётчик с настроенными целями: отправка формы, клик по телефону, открытие калькулятора, переход в мессенджер. Даёт понимание, откуда приходят клиенты и какие страницы конвертируют лучше всего.

Q3
2025

Telegram-бот для уведомлений

Клиент подписывается на уведомления через Telegram и получает автоматические сообщения о каждом изменении статуса груза: «Принят на склад», «Вышел в рейс», «На таможенном оформлении», «Доставлен». Также бот отвечает на типовые вопросы.

Q3
2025

Реальный трекинг контейнеров через SeaRates API

Подключение к SeaRates или ShipsGo API: клиент вводит номер контейнера или коносамента и видит реальное положение судна, ETA, портовые события. Покрытие: Maersk, MSC, CMA CGM, FESCO, Cosco, Evergreen и другие линии.

Q4
2025

Электронный документооборот (ЭДО)

Интеграция с операторами ЭДО (Диадок, СБИС): клиент подписывает накладные, акты и счёт-фактуры в личном кабинете без распечатки. Сокращает цикл закрытия сделки с 5–7 дней до 1 дня.

Q4
2025

Интеграция с 1С: Бухгалтерия / 1С: Управление торговлей

Автоматическая выгрузка счетов и актов из 1С в личный кабинет клиента. Клиент видит финансовые документы без запросов бухгалтерии. Для команды: исключает ручную выгрузку PDF из 1С.

Q1
2026

Мобильное приложение (PWA)

Прогрессивное веб-приложение: клиент добавляет сайт на экран смартфона и получает push-уведомления о грузах. Работает без AppStore/Google Play. Стоимость разработки в 3–4 раза ниже нативного приложения.

Q2
2026

Open API для партнёров и клиентов

REST API позволит крупным клиентам интегрировать данные Pacific Star напрямую в свою ERP/TMS-систему: создавать заявки, получать статусы, скачивать документы программно. Аналог FESCO и TransContainer.
