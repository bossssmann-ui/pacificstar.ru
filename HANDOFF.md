## Рекомендуемый порядок для нового агента

### Шаг 0 — Подготовка (15 минут)

1. **Прочитать** [ROADMAP.md](ROADMAP.md) — там фазы, статусы задач и критерии готовности.
2. **Проверить ветку и PR:**
   - Рабочая ветка: `cursor/site-roadmap-and-phase1-aa30`
   - Draft PR: https://github.com/bossssmann-ui/pacificstar.ru/pull/213
   - Если PR ещё не в `main` — работать от этой ветки или смержить PR перед стартом.
3. **Уточнить у заказчика доступы** (без них часть фаз заблокирована):

   | Доступ | Нужен для |
   |--------|-----------|
   | Timeweb Node.js / VPS | Фаза 0 — формы на production |
   | SMTP (Яндекс / Mail.ru) | Письма с заявок |

4. **Локальный запуск** (для проверок):
   ```bash
   npm install
   npm run validate    # html-validate
   npm start           # server.js на :3000 (нужен .env для SMTP)
   ```

## Настройка автономной работы агента

Цель: вы один раз выдаёте доступы и правила — агент идёт **задача за задачей** по ROADMAP, коммитит, пушит, обновляет PR. Зовёт вас только при **серьёзных** вопросах (см. таблицу ниже).

### Что выдать агенту один раз

#### 1. GitHub (обязательно)

| Что | Зачем | Где настроить |
|-----|-------|---------------|
| Доступ к репо `bossssmann-ui/pacificstar.ru` | Ветки, коммиты, PR | GitHub → Settings → Collaborators или Cloud Agent |
| Право пушить ветки `cursor/*-aa30` | Без ожидания вас | У агента / Cloud Agent |
| **Разрешение мержить в `main`** | Автодеплой на production | Явно в промпте: «можешь мержить PR после validate» **или** «мержу только я» |

> Если агент **не** мержит в `main` — он всё равно может закрывать задачи в ветках и draft PR; на сайт изменения попадут после вашего merge.

#### 2. GitHub Secrets (для деплоя статики — уже есть)

| Secret | Назначение |
|--------|------------|
| `TIMEWEB_HOST` | FTP/SSH хостинг |
| `TIMEWEB_USER` | Логин |
| `TIMEWEB_PASSWORD` | Пароль |

Проверка: Actions → [tilda-deploy.yml](.github/workflows/tilda-deploy.yml) → Run workflow.

#### 3. Секреты для backend и интеграций (добавить в GitHub Secrets или Cursor Environment)

Агенту нужны **имена** секретов — не вставляйте пароли в чат. Лучше: Cursor Cloud → Environment secrets или GitHub Actions secrets.

| Secret (рекомендуемое имя) | Переменная в `.env` | Фаза | Обязательно |
|----------------------------|---------------------|------|-------------|
| `SMTP_HOST` | `SMTP_HOST` | 0 | Да |
| `SMTP_USER` | `SMTP_USER` | 0 | Да |
| `SMTP_PASS` | `SMTP_PASS` | 0 | Да |
| `SMTP_PORT` | `SMTP_PORT` | 0 | Нет (465 по умолчанию) |
| `MAIL_FROM` | `MAIL_FROM` | 0 | Нет |
| `PS_YM_ID` | в HTML / config | 2 | Да для Метрики |
| `PS_GA_ID` | опционально | 2 | Нет |
| `AMOCRM_WEBHOOK_URL` или `AMOCRM_*` | server / main.js | 2 | Да для CRM |
| `NODE_DEPLOY_*` | SSH/host для API | 0 | Если API не на Timeweb shared |

Шаблон локального `.env`: [.env.example](.env.example). Файл `.env` **никогда** не коммитить.

#### 4. Доступ к серверу для `server.js` (Фаза 0)

Выберите **один** вариант и передайте агенту:

| Вариант | Что дать агенту |
|---------|-----------------|
| **A. Node на Timeweb** | SSH или панель + путь к приложению + как проксировать `/api/*` |
| **B. Отдельный VPS** | SSH-ключ или host/user + PM2/systemd |
| **C. Serverless** (Railway, Render) | API URL + токен деплоя |
| **D. Пока нет backend** | Написать в промпте: «Фазу 0 пропусти, делай 2.3–2.5 и Фазу 3» |

Без варианта A/B/C агент **не сможет** починить формы на production — только подготовить код.

#### 5. Правила для агента (вставить в промпт)

```
Режим: автономный.
- Работай по HANDOFF.md и ROADMAP.md задача за задачей.
- После каждой задачи: validate → commit → push → обновить ROADMAP (⬜→✅) → обновить PR.
- Не спрашивай разрешения на рутинные правки в рамках текущей задачи ROADMAP.
- Мерж в main: [РАЗРЕШЁН / ТОЛЬКО DRAFT PR — реши сам в промпте].
- Зови меня ТОЛЬКО если сработало условие из таблицы «Когда звать заказчика».
```

| Ситуация | Пример | Действие агента без вас |
|----------|--------|-------------------------|

| ❌ **Звать** | Удаление страниц, смена юрлица, цены в договоре | Не трогать |
| ❌ **Звать** | Конфликт PR / чужие незакоммиченные изменения | Остановиться |
| ❌ **Звать** | Production сломан после деплоя | Откат + сообщение |
| ✅ **Не звать** | Вёрстка, SEO, FAQ, i18n, баги UI | Делать самому |
| ✅ **Не звать** | html-validate, cache-bump, noscript, JSON-LD | Делать самому |
| ✅ **Не звать** | Draft PR, новая ветка `cursor/...-aa30` | Делать самому |
| ✅ **Не звать** | Нет OG-картинки — сделать временную из logo | Делать самому или placeholder |
| ✅ **Не звать** | Тестовая заявка на staging / localhost | Делать самому |

### Минимальный набор «агент работает сам 80% времени»

Если хотите **минимум вовлечения**, выдайте:

1. ✅ Доступ к GitHub-репо (агент пушит ветки)
2. ✅ Решение: **«мержишь в main сам после `npm run validate`»** или **«только draft PR»**
3. ⬜ SMTP в secrets (для Фазы 0) — *или явно «Фазу 0 отложи»*
4. ⬜ `PS_YM_ID` (число счётчика) — для Метрики
5. ⬜ AmoCRM webhook URL — для CRM

**Без п.3–6** агент спокойно закроет: мобильный телефон, SEO-тексты, i18n, UI-баги, остатки Фазы 1, подготовку кода под CRM/Метрику (с placeholder для ID).
